Introduction

Akasha is an open-source, privacy-first personal assistant designed to run entirely on your local machine. No internet connection required for core functionality — your data never leaves your device.

Akasha is powered by a flexible skill system: each capability (weather, reminders, file management, etc.) is a self-contained skill you can install, update or remove. The core is minimal and fast; skills add exactly what you need.

💡
Quick start: Jump straight to Installation if you just want to get running.

Key Principles

  • Privacy by default — no telemetry, no cloud sync, no user accounts.
  • Extensible — install skills to add new capabilities.
  • Fast — local processing means zero network latency.
  • Open source — MIT licensed, community-driven.

Installation

Requirements

  • OS: Windows 10+, macOS 11+, or Ubuntu 20.04+
  • Ollama (optional) for local LLM models; otherwise configure a cloud provider (OpenRouter, OpenAI) during init.

Download

Go to the Releases page (or GitHub Releases). Download the archive for your OS (Windows / Linux / macOS), then extract it to a folder (e.g. C:\Akasha or ~/Akasha).

You get the akasha (or akasha.exe), akasha-daemon, akasha-tui executables and a docs folder with the user guide.

First run (Linux / macOS)

chmod +x akasha akasha-daemon akasha-tui
./akasha init
./akasha start
./akasha tui   # terminal interface

First run (Windows)

.\akasha.exe init
.\akasha.exe start
.\akasha.exe tui   # terminal interface

Run akasha start from the folder where you extracted the archive so the in-app Doc tab can show the user guide.

First Run

akasha init runs an interactive setup: LLM provider (Ollama, OpenRouter, OpenAI), vault, and optional channels (Telegram, Slack, Discord). Use akasha init --defaults for minimal setup without prompts.

Then start the daemon and open the interface:

akasha start      # background daemon (auto-restart on crash)
akasha tui        # terminal UI: Chat, Router, Doc, Tasks, Calendar, Memory

If you have the desktop app (Akasha UI), launch it; it connects to the daemon on port 3876.

Commands Reference

Daemon

akasha start              # Start daemon (auto-restart)
akasha start --foreground # Run in foreground (logs in terminal)
akasha stop               # Stop daemon

Setup & diagnostic

akasha init              # Interactive setup
akasha init --defaults   # Minimal setup, no prompts
akasha doctor            # Check system and daemon
akasha doctor --fix      # Create data_dir and minimal config if missing
akasha doctor --advice   # Diagnostic advice (daemon must be running)
akasha paths             # Show data_dir and config file paths

Vault (secrets)

akasha vault list        # List stored keys
akasha vault set KEY [value]  # Store a secret
akasha vault get KEY     # Show value (use with care)
akasha vault delete KEY  # Remove a key

Config & router

akasha config models routes   # Models per category
akasha config models set CATEGORY PROVIDER MODEL
akasha config env list   # List env vars in akasha.env
akasha router metrics    # LLM router metrics

Interfaces

akasha tui   # Terminal UI (Chat, Router, Doc, Tasks, Calendar, Memory)

Update check

akasha update check   # Check for a new version (download from Releases)

Configuration

All config files live in the data directory. Run akasha paths to see its path (e.g. C:\Users\…\AppData\Local\akasha on Windows, ~/.local/share/akasha on Linux/macOS).

Main files

  • llm_router.yaml — LLM providers (Ollama, OpenAI, OpenRouter) and models per task type.
  • tools_policy.yaml — Tool permissions: file paths, allowed commands, web domains, skill-install hosts.
  • akasha.env — Persistent env vars (also editable via akasha config env set).
  • connectors.env — Telegram, Slack, Discord activation.

akasha init or akasha doctor --fix creates the data_dir and minimal config files if missing.

Interfaces

The TUI (akasha tui) has tabs: Chat, Router, Doc, Tasks, Calendar, Memory. The desktop app (if installed) connects to the same daemon and supports attachments (images, documents) and user RAG in Settings.

Using Skills

Ask the agent in chat to install a skill from a URL, e.g. « Install the bankr skill from https://github.com/BankrBot/skills/tree/main/bankr ». The agent will download it and reload skills.

To allow other hosts (GitLab, custom HTTPS), edit tools_policy.yaml in the data_dir and set allowed_skill_install_hosts (e.g. ["*"] for any HTTPS).

If you add or change skill files manually in the data_dir, type /skills reload in the chat to reload without restarting the daemon.

Slash commands (in chat)

In the chat (TUI or web), messages starting with / are commands:

/help, /?        # Help
/status          # Daemon status
/doctor          # Diagnostic
/skills reload   # Reload skills after adding or editing files
/restart         # Restart daemon (when run with supervisor)
/models list     # Models by category
/config list     # Env vars (akasha.env)
/vault list      # Vault key names

Use the CLI for vault write/delete: akasha vault set KEY value, akasha vault delete KEY.

Update API

Akasha checks the following endpoint on startup (if auto-update is enabled):

GET https://azerothl.github.io/Akasha_app/api/latest.json

Response format

{
  "version": "0.2.0",
  "release_date": "2026-03-10",
  "release_notes_url": "https://azerothl.github.io/Akasha_app/releases.html",
  "download_url": "https://github.com/azerothl/Akasha_app/releases/tag/v0.2.0",
  "changelog": "Calendar improvements, release and deploy workflow fixes, Tauri icons.",
  "min_supported_version": "0.5.0"
}

Version comparison

Akasha uses semantic versioning (semver). An update is triggered when version in the response is greater than the installed version.

Run akasha update check to see if a newer version is available; the command prints the download link.
