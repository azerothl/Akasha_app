Introduction

Akasha is an open-source, privacy-first personal assistant designed to run entirely on your local machine. No internet connection required for core functionality — your data never leaves your device.

Akasha is powered by a flexible skill system: each capability (weather, reminders, file management, etc.) is a self-contained skill you can install, update or remove. The core is minimal and fast; skills add exactly what you need.

💡
Quick start: Jump straight to Installation if you just want to get running.

Key Principles

  • Privacy by default — no telemetry, no cloud sync, no user accounts.
  • Extensible — install skills to add new capabilities.
  • Fast — local processing means zero network latency.
  • Open source — MIT licensed, community-driven.

Installation

Requirements

  • OS: Windows 10+, macOS 11+, or Ubuntu 20.04+
  • RAM: 512 MB minimum (2 GB recommended)
  • Disk: 100 MB for the base install

Linux / macOS

# Download the latest installer
curl -L https://azerothl.github.io/Akasha_app/api/latest.json | python3 -c "import sys,json; print(json.load(sys.stdin)['download_url'])"

# Make executable and run
chmod +x akasha-installer.sh
./akasha-installer.sh

Windows

Download akasha-setup.exe from the Releases page and run the installer.

Verify installation

akasha --version
# Akasha v1.0.0
ℹ
After installation, Akasha adds itself to your PATH. Open a new terminal to use the akasha command.

First Run

Launch Akasha for the first time:

akasha

A configuration wizard will walk you through:

  1. Choosing your default language
  2. Selecting a UI theme (dark / light)
  3. Optionally installing starter skill packs
  4. Enabling or disabling auto-update checks

Interactive mode

Start the interactive assistant shell:

akasha shell
# ❯

One-shot mode

Run a single query and exit:

akasha "What time is it in Tokyo?"
# 🕑  Tokyo, Japan — 09:42 JST (UTC+9)

Commands Reference

Core commands

akasha                  # Start interactive shell
akasha shell            # Alias for interactive mode
akasha "query"          # One-shot query
akasha --version        # Print version
akasha --help           # Print help

Skill management

akasha install <skill>   # Install a skill from the library
akasha uninstall <skill> # Remove an installed skill
akasha skills            # List installed skills
akasha skills --all      # List all available skills
akasha update            # Update all installed skills

Configuration

akasha config            # Open interactive config
akasha config --list     # Print current configuration
akasha config set key value  # Set a config value

Update management

akasha check-update      # Check for a new version
akasha upgrade           # Download and install the latest version

Configuration

Configuration is stored in ~/.akasha/config.json. You can edit it directly or use the CLI.

Default config

{
  "language": "en",
  "theme": "dark",
  "auto_update": true,
  "update_channel": "stable",
  "startup_message": true,
  "log_level": "info",
  "skills_dir": "~/.akasha/skills"
}

Key settings

  • language — Interface language code (e.g. "en", "fr", "es")
  • theme — UI theme: "dark" or "light"
  • auto_update — Whether Akasha checks for updates on startup
  • update_channel — "stable" or "beta"
  • log_level — Logging verbosity: "error", "warn", "info", "debug"
⚠
Manual edits: Restart Akasha after manually editing config.json for changes to take effect.

Themes

Akasha ships with two built-in themes and supports custom CSS theming.

Switching themes

akasha config set theme dark   # Dark theme (default)
akasha config set theme light  # Light theme

Custom themes

Place a CSS file in ~/.akasha/themes/my-theme.css, then:

akasha config set theme my-theme

Theme CSS variables you can override:

:root {
  --bg-primary:   #0d0d14;
  --bg-secondary: #12121e;
  --purple:       #8b5cf6;
  --cyan:         #06b6d4;
  --text-primary: #f1f5f9;
  /* ... */
}

Using Skills

Skills are the building blocks that give Akasha its capabilities. The base install includes a minimal core; you install skills for the features you need.

Installing skills

akasha install weather
# ↓  Downloading weather skill v1.2.0…
# ✓  Installed successfully

Listing skills

akasha skills
# Installed skills:
#   weather     v1.2.0  ✓ up to date
#   calculator  v2.0.0  ✓ up to date
#   timer       v1.0.3  ✓ up to date

Updating skills

akasha update           # Update all skills
akasha update weather   # Update a specific skill

Uninstalling

akasha uninstall weather
💡
Browse the full Skills Library to discover what's available.

Skills API

Build your own skill by creating a directory with this structure:

my-skill/
├── skill.json      # Metadata
├── index.js        # Entry point
└── README.md       # Documentation

skill.json

{
  "id": "my-skill",
  "name": "My Skill",
  "version": "1.0.0",
  "description": "What this skill does",
  "author": "Your Name",
  "category": "utility",
  "tags": ["example"],
  "entry": "index.js",
  "permissions": []
}

index.js — minimal example

module.exports = {
  // Patterns that trigger this skill
  patterns: [/hello/i, /hi akasha/i],

  // Called when a pattern matches
  async handle(query, context) {
    return `Hello! You said: "${query}"`;
  }
};

Available permissions

  • network — Allow HTTP requests
  • filesystem — Read/write local files
  • clipboard — Read/write clipboard
  • notifications — Show system notifications
ℹ
All declared permissions are shown to the user before installation. Skills without network permission cannot make HTTP calls.

Update API

Akasha checks the following endpoint on startup (if auto-update is enabled):

GET https://azerothl.github.io/Akasha_app/api/latest.json

Response format

{
  "version": "1.0.0",
  "release_date": "2025-01-15",
  "release_notes_url": "https://azerothl.github.io/Akasha_app/releases.html",
  "download_url": "https://github.com/azerothl/Akasha_app/releases/latest",
  "changelog": "Brief description of changes.",
  "min_supported_version": "0.5.0"
}

Version comparison

Akasha uses semantic versioning (semver). An update is triggered when version in the response is greater than the installed version.

Disabling update checks

akasha config set auto_update false

Contributing

Contributions are welcome! Here's how to get started:

Development setup

git clone https://github.com/azerothl/Akasha_app.git
cd Akasha_app
npm install
npm run dev

Contribution types

  • 🐛 Bug reports — open an issue
  • ✨ Feature requests — discuss in Discussions
  • 🔧 Pull requests — fork, branch, code, PR
  • 🧩 Skills — build and share skills
  • 📖 Docs — improve documentation

Code style

  • Use 2-space indentation
  • Prefer const and let over var
  • Write descriptive commit messages
  • Add tests for new features
💡
Please read CONTRIBUTING.md in the repository before submitting a PR.
